## 3. AgentOS 3.0 Hierarchy: Visualizing Intelligence

Our visual hierarchy is a direct reflection of the "Paradigm Shift" from fixed to elastic. We use our atomic shapes to build a clear, recognizable caste system for our 300+ agents.

### Level 1: MASTER (The Orchestrator)

**Role:** The strategic conductor, the "central intelligence" that connects all dots.

**Visual Signature:** **The Halo**
A central Insight Node (Circle) surrounded by a concentric, orbiting ring.

**Visual Specs:**
- Core circle: 48px diameter, solid fill in Tenant Color
- Halo ring: 72px diameter, 2px stroke, 60% opacity
- Optional: 4-6 small connection nodes (8px circles) on orbital path

**Usage:** High-stakes, multi-domain orchestration (Mode 3)

### Level 2: EXPERT (The Domain Specialist)

**Role:** The primary user-facing consultant with deep domain knowledge.

**Visual Signature:** **Geometric Personhood**
A distinct "Head" (Circle) resting on a solid, domain-specific "Body" shape (Square, Triangle, etc.), flooded in the Tenant Color.

**Visual Specs:**
- Head: 32px circle, solid Tenant Color
- Body: 48px width geometric shape (square, triangle, trapezoid)
- Total height: ~80px
- 2px stroke outline in Neutral-900

**Usage:** The primary cards in the Agent Grid

### Level 3: SPECIALIST (The Focused Capability)

**Role:** A single-purpose sub-expert spawned for a specific task.

**Visual Signature:** **Pure Focus**
A single, solid geometric primitive (e.g., just a Triangle or Diamond) without a "head," in a lighter tint of the parent color.

**Visual Specs:**
- Single shape: 32px, no "head"
- Color: 40% opacity of parent Tenant Color
- Stroke: 2px, Neutral-900

**Usage:** Nested in trace views or panels

### Level 4: WORKER (The Invisible Infrastructure)

**Role:** The background execution team handling repetitive tasks.

**Visual Signature:** **Functional Structure**
Minimal, outlined grids or stacks. Colorless (Neutral-900) and purely structural.

**Visual Specs:**
- Grid pattern: 3Ã—3 or 2Ã—4 squares
- Stroke: 1.5px, Neutral-900
- No fill, pure outline
- Size: 24-32px

**Usage:** Only visible in logs or execution traces

### Level 5: TOOL (The Precision Instrument)

**Role:** A passive utility or API connector.

**Visual Signature:** **Mechanical Outline**
Hollow, outlined geometric forms that imply mechanics, like a hexagon with a central dot or a connector plug.

**Visual Specs:**
- Geometric outline: Hexagon, plug, gear
- Stroke: 2px, Neutral-900
- Center dot: 4px, Neutral-600
- Size: 24px

**Usage:** Listed in Tools Library or execution logs

## 5. Typography

### 5.1 Primary Typeface: Inter

**Why Inter:**
Geometric, modern, universally legible. Designed for screens. Open-source.

**Weight Hierarchy:**
- **Inter ExtraBold (800)**: Hero titles, "VITAL" logo text, primary headlines
  - Size: 48-72px for heroes, 24-32px for section titles
  - Use: Marketing materials, homepage hero, major announcements

- **Inter Bold (700)**: Section headers, Agent Roles, card titles
  - Size: 18-24px
  - Use: UI headers, agent names, primary navigation

- **Inter Medium (500)**: UI labels, buttons, metadata
  - Size: 14-16px
  - Use: Form labels, button text, secondary navigation

- **Inter Regular (400)**: Body copy, descriptions
  - Size: 14-16px (body), 12-14px (captions)
  - Use: Paragraphs, agent descriptions, help text

### 5.2 Technical Typeface: JetBrains Mono

**Why JetBrains Mono:**
Monospaced, excellent readability, designed for code.

**Usage:**
- Code blocks and syntax highlighting
- Agent execution logs and traces
- System metadata and IDs
- Technical documentation
- API responses

**Weights:**
- **Regular (400)**: Standard code display
- **Bold (700)**: Emphasis in code, active line

### 5.3 Typography Principles

1. **Large, spacious titles** - Generous whitespace around headlines
2. **Tight vertical rhythm** - Line-height: 1.4-1.6 for body, 1.2 for headers
3. **Generous line height** - Never less than 1.4 for readability
4. **Clear visual hierarchy** - Size + weight create scannable content
5. **Bold for meaning, not noise** - Use weight to emphasize, not decorate

## 6. Illustration & Iconography System

Our illustrations are not decorations; they are visualizations of abstract concepts like "elasticity," "compounding knowledge," and "orchestration."

### 6.1 The "Hero" Illustration Style: Elastic Networks

**Concept:** A dynamic, interconnected web of atomic shapes representing the VITAL network.

**Execution:**
- Complex composition of outlined cubes, circles, and connecting lines
- 2px stroke weight throughout
- Key data nodes highlighted with **solid Purple accents**
- Shows intelligence flow through the system

**Usage:**
- Homepage hero sections
- Marketing landing pages
- Pitch decks and presentations
- Product overview diagrams

### 6.2 Spot Illustration & Icon Style: Atomic Concepts

**Concept:** Simple, clever combinations of 2-3 atomic shapes to communicate a single idea.

**Execution:**
- 2px Neutral-900 stroke
- **One** shape filled with accent color (usually Purple) to create focal point
- 24Ã—24 or 48Ã—48 canvas
- Minimal fills, maximum clarity

**Icon Categories:**

**Navigation Icons:**
- Home, Dashboard, Services, Library, Settings, Admin

**Feature Icons:**
- Ask Expert, Ask Panel, Workflows, JTBD, Knowledge, Tools

**Status Icons:**
- Active, Testing, Development, Error, Success, Processing

**Agent Type Icons:**
- Master, Expert, Specialist, Worker, Tool

### 6.3 Icon Grammar Rules

1. **Construction**: Built strictly from Atomic Shapes (Circle, Square, Triangle, Line, Diamond)

3. **No Literal Metaphors**: No "magnifying glasses" for search unless abstractly constructed
4. **Color**: Neutral (#1A1A1A) stroke, accent fill on ONE atomic shape for meaning
5. **Scalability**: Must work at 16px, 24px, 48px without losing clarity

### 6.4 Dashboard Icon Concept: "Grid & Insight" Family

**2Ã—2 Grid**: Represents Modules/Data
**Radiant Node**: Represents Intelligence/Insight

## 7. UI & Layout: Warm Modernism Applied

### 7.1 Philosophy: "Warm Modernism"

Our UI is a canvas for intelligent content. Clean, generous with space, using atomic system to guide users.

**Core Principles:**
- **Backgrounds**: Always `#FAF8F1` (Warm Ivory), never stark white (`#FFFFFF`)
- **Spacing**: Generous, 8pt grid system (8, 16, 24, 32, 40, 48, 64, 80px)
- **Cards**: Soft rounded corners (8px radius), subtle borders (Neutral-200), minimal shadows
- **Typography**: Inter for UI, JetBrains Mono for technical content
- **Composition**: Let geometric avatars and illustrations breathe with negative space

### 7.2 Layout Grid System

**8pt Grid Foundation:**
- Base unit: 8px
- Common spacings: 8, 16, 24, 32, 48, 64px
- Component padding: 16-24px
- Section margins: 48-64px
- Page margins: 64-80px

### 7.3 Component Specifications

#### Agent Cards

**Standard Agent Card:**
- **Dimensions**: 320px width, auto height
- **Container**: `#FAF8F1` background, 1px `#E8E5DC` border, 12px radius
- **Padding**: 24px
- **Components**:
  - Avatar: 80px height, centered or left-aligned
  - Title: Inter Bold 700, 20px
  - Description: Inter Regular 400, 14px, 2-3 lines max
  - Tags: Pill badges (8px radius, Tenant Color fill)
  - Footer: Divider line + action buttons

**Compact Agent Card:**
- **Dimensions**: 280px width, auto height
- **Avatar**: 56px height
- **Title**: Inter Bold 700, 16px
- **Single action button**

#### Data Visualization

**Chart Styling:**
- **Background**: Transparent or `#FAF8F1`
- **Grid lines**: `#E8E5DC`, 1px stroke
- **Axes**: `#1A1A1A`, Inter Regular 400, 12px
- **Data lines**: 2px stroke, Tenant Colors
- **Data points**: 6px circles, filled with Tenant Color
- **Tooltips**: White background, 1px `#E8E5DC` border, 8px radius

## 9. Voice & Tone

### 9.1 Primary Tone

**Clear, Precise, Intelligent.**
We speak like a senior partner, not a chatbot.

**Characteristics:**
- Professional yet approachable
- Confident without arrogance
- Technical without jargon
- Helpful without being patronizing

### 9.2 Writing Principles

1. **Clarity First**: Simple, direct language
2. **Active Voice**: "VITAL orchestrates..." not "Orchestration is provided by..."
3. **Specific Over Generic**: "136 healthcare expert agents" not "many AI assistants"
4. **Value-Focused**: Lead with benefits, follow with features
5. **Human-Centric**: Technology serves people, not replaces them

### 9.3 Terminology Standards

**âœ… Use These Terms:**

- **Automation**: Implies replacement; we amplify, not replace
