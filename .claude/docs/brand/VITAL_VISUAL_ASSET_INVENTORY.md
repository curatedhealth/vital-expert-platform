### 4.2 Database Schema for Asset Management

**Table: `icons`** (Extend existing from migration 20250927050000)

```sql
CREATE TABLE icons (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),

    -- Asset Identity
    name VARCHAR(255) UNIQUE NOT NULL,              -- e.g., 'super_orchestrator'
    display_name VARCHAR(255) NOT NULL,             -- e.g., 'Super Orchestrator'
    category VARCHAR(50) NOT NULL,                  -- 'super_agent', 'icon', 'avatar'
    subcategory VARCHAR(100),                       -- 'navigation', 'feature', 'status'

    -- Asset Location
    file_path TEXT NOT NULL,                        -- relative path in storage
    file_url TEXT NOT NULL,                         -- public URL
    svg_content TEXT,                               -- actual SVG markup (optional cache)

    -- Taxonomy & Metadata
    tags TEXT[] DEFAULT '{}',                       -- searchable tags
    persona_type VARCHAR(50),                       -- for avatars: 'expert', 'foresight', etc.
    department VARCHAR(100),                        -- for avatars: 'medical_affairs', etc.
    color_variant VARCHAR(50),                      -- 'black', 'purple', 'full_color'

    -- Visual Specifications
    canvas_size INTEGER DEFAULT 256,                -- 256x256 for avatars, 24x24 for icons
    stroke_weight INTEGER DEFAULT 2,                -- 2px for icons, 4-6px for super agents
    primary_color VARCHAR(7),                       -- hex color code

    -- Agent Mapping (for avatars)
    agent_level INTEGER,                            -- 1=Master, 2=Expert, 3=Specialist
    tenant_affinity VARCHAR(100)[],                 -- ['expert', 'pharma', 'medical']
    capability_indicators VARCHAR(100)[],           -- ['halo', 'radiant', 'diamond']

    -- Status & Governance
    is_active BOOLEAN DEFAULT true,
    is_public BOOLEAN DEFAULT true,
    usage_count INTEGER DEFAULT 0,                  -- track popularity

    -- Timestamps
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- Indexes for fast queries
CREATE INDEX idx_icons_category ON icons (category);
CREATE INDEX idx_icons_subcategory ON icons (subcategory);
CREATE INDEX idx_icons_persona_department ON icons (persona_type, department);
CREATE INDEX idx_icons_tags ON icons USING GIN (tags);
CREATE INDEX idx_icons_tenant_affinity ON icons USING GIN (tenant_affinity);
CREATE INDEX idx_icons_agent_level ON icons (agent_level);
```

### 4.3 Icon-to-Feature Mapping Strategy

CREATE TABLE icon_feature_mappings (

    icon_id UUID REFERENCES icons(id) ON DELETE CASCADE,

    -- Priority

## 5. Agent-Avatar Intelligent Mapping Algorithm

### 5.1 Mapping Criteria

**Multi-factor scoring system for optimal agent-avatar pairing:**

```
Avatar Score = (
    tier_match * 0.30 +
    domain_match * 0.25 +
    persona_match * 0.20 +
    tenant_affinity * 0.15 +
    visual_harmony * 0.10
)
```

**Factor Definitions:**

1. **Tier Match (30%)**
   - Agent tier (tier_1, tier_2, tier_3) â†’ Avatar level mapping
   - Tier 3 = Expert personas, bold colors, halo indicators
   - Tier 2 = Specialist personas, refined shapes
   - Tier 1 = Startup personas, simple forms

2. **Domain Match (25%)**
   - Agent's `department` field â†’ Avatar `department` taxonomy
   - Direct match: 100%, related match: 70%, unrelated: 30%

3. **Persona Match (20%)**
   - Agent's role complexity â†’ Avatar `persona_type`
   - Strategic/Leadership â†’ expert/foresight
   - Clinical/Execution â†’ medical/pharma
   - Foundational/Support â†’ startup

4. **Tenant Affinity (15%)**
   - Agent's `tenant_id` or `business_function` â†’ Avatar `tenant_affinity` array
   - Checks if avatar's color palette aligns with tenant brand

5. **Visual Harmony (10%)**
   - Capability indicators match agent's `capabilities` array
   - Ensures avatar visual cues align with agent functionality
