# L4 Worker Prompt Starter Templates

## Overview

L4 Workers are cross-functional agents that perform repetitive, standardized tasks. They can be used by agents across all departments for common operational needs.

## Key Characteristics

- **Cross-functional**: Available to all departments
- **Repetitive tasks**: Standardized, repeatable operations
- **Process-oriented**: Follow defined procedures
- **Automation-ready**: Suitable for workflow automation

## Template Patterns

### Tracking/Monitoring
```
"Track [items/status] and [report/flag] [findings] requiring [action]"
```

**Variables:**
- `[items]`: approvals, deadlines, tasks, documents, milestones
- `[report type]`: status update, escalation, summary
- `[findings]`: overdue items, bottlenecks, compliance gaps
- `[action]`: attention, review, escalation

**Examples:**
```
"Track pending approvals for all materials in the review queue and flag overdue items"
"Track project milestones and report status deviations requiring management attention"
```

---

### Processing/Formatting
```
"Process the [input] according to [standard/template] and [output result]"
```

**Variables:**
- `[input]`: data, documents, reports, submissions
- `[standard]`: company template, regulatory format, style guide
- `[output]`: formatted document, processed data, validated output

**Examples:**
```
"Process the clinical data tables according to CSR template standards"
"Format the regulatory submission documents following eCTD structure requirements"
```

---

### Generation/Compilation
```
"Generate a [report/output] showing [metrics/data] for [period/scope]"
```

**Variables:**
- `[report type]`: status report, metrics dashboard, summary document
- `[metrics]`: KPIs, compliance rates, activity counts
- `[scope]`: weekly, monthly, quarterly, by team, by project

**Examples:**
```
"Generate a weekly status report showing task completion rates by team member"
"Compile a quarterly metrics report on document review cycle times"
```

---

### Validation/Checking
```
"Validate [input] against [criteria] and flag [issues] for [action]"
```

**Variables:**
- `[input]`: documents, data, references, formatting
- `[criteria]`: naming conventions, compliance requirements, quality standards
- `[issues]`: errors, non-compliance, missing elements
- `[action]`: correction, review, approval

**Examples:**
```
"Validate file naming conventions across the project folder and flag non-compliant documents"
"Check document formatting against corporate template and highlight deviations"
```

---

### Organization/Management
```
"Organize [items] according to [structure] and [maintain/update] [records]"
```

**Variables:**
- `[items]`: files, documents, records, archives
- `[structure]`: folder hierarchy, naming convention, classification system
- `[records]`: index, metadata, audit trail

**Examples:**
```
"Organize project files according to TMF structure and update the document index"
"Archive completed project documents and maintain folder structure compliance"
```

## Cross-Functional L4 Worker Templates

### Chart Generator
1. "Create a bar chart showing enrollment progress by site from the provided data"
2. "Generate a timeline visualization of project milestones and dependencies"
3. "Build a pie chart displaying budget allocation by category"
4. "Create a line graph showing safety trends over the study period"

### Dashboard Builder
1. "Build a project status dashboard showing key milestones, risks, and resource utilization"
2. "Create an executive summary dashboard with KPIs and trend indicators"
3. "Generate a team performance dashboard with activity metrics and targets"
4. "Build a compliance tracking dashboard showing audit findings and CAPA status"

### Report Formatter
1. "Format the quarterly report according to corporate template standards"
2. "Reformat the data tables to match regulatory submission requirements"
3. "Apply consistent styling to the presentation following brand guidelines"
4. "Format the meeting minutes using the approved company template"

### Status Report Generator
1. "Generate a weekly project status report with milestones achieved and upcoming deliverables"
2. "Create a monthly activity summary showing completed tasks and resource utilization"
3. "Compile a risk status report highlighting new, changed, and closed risks"
4. "Generate an approval status report showing pending items and bottlenecks"

### Document Generator
1. "Generate a meeting agenda from the provided discussion topics and attendee list"
2. "Create a project charter document using the standard template and provided inputs"
3. "Generate a compliance checklist based on the regulatory requirements matrix"
4. "Create a training completion certificate from participant and course data"

### Data Validator
1. "Validate the uploaded data file against the expected schema and flag errors"
2. "Check the reference list for completeness and correct formatting"
3. "Validate date fields for logical consistency and proper format"
4. "Verify numeric data falls within expected ranges and flag outliers"

### PowerPoint Generator
1. "Create a presentation from the provided outline and data points"
2. "Build slides from the meeting notes using the standard template"
3. "Generate a data presentation from the Excel tables provided"
4. "Create a summary presentation from the report content"

### Meeting Notes Generator
1. "Generate formatted meeting notes from the provided transcript"
2. "Create action item list with owners and deadlines from meeting discussion"
3. "Summarize key decisions and next steps from the meeting recording"
4. "Generate a meeting summary email for distribution to attendees"
