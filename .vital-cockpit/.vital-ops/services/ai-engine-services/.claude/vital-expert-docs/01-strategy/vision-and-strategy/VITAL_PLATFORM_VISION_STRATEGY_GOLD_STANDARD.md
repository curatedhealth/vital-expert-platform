### 10.3 Multi-Tenant Architecture

**4 Tenant Types**:

| Tenant Type | Purpose | Example | Data Isolation |
|-------------|---------|---------|----------------|
| **Platform** | VITAL's own data | Agent registry, templates | Shared across all |
| **Client** | Customer organization | Pharma Company X | Isolated (RLS) |

| **Industry** | Industry-wide shared data | Regulatory guidelines | Shared within industry |

**Multi-Tenant Data Model**:
```sql
-- Every table has tenant_id
CREATE TABLE consultations (
  id UUID PRIMARY KEY,
  tenant_id UUID REFERENCES tenants(id),  -- Multi-tenant key
  user_id UUID REFERENCES users(id),
  agent_id UUID REFERENCES agents(id),
  created_at TIMESTAMP,
  ...
);

-- Row-Level Security (RLS) enforces isolation
CREATE POLICY "Tenant isolation"
  ON consultations
  FOR ALL
  USING (tenant_id = auth.jwt() ->> 'tenant_id');
```

**Benefits**:
- âœ… **Data isolation**: Each customer's data completely separated
- âœ… **Shared infrastructure**: All customers on same servers (cost efficiency)
- âœ… **Compliance**: Meets enterprise security requirements
