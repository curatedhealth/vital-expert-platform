## Agent Attributes (15 Dimensions)

Each agent is characterized by **15 distinct attributes**:

### Core Identifiers
- **unique_id**: Slug (e.g., `msl-expert-agent`)
- **name**: Display name (e.g., "Medical Science Liaison Expert")
- **slug**: URL-friendly identifier
- **domain**: Primary function (e.g., "Medical Affairs")
- **subdomain**: Specific area (e.g., "Field Medical")

### Profile
- **description**: 2-3 sentence overview
- **capabilities**: Array of 5-15 specific skills
- **expertise_areas**: Array of knowledge domains
- **years_of_experience**: Typical experience level (5-20+ years)
- **seniority_level**: Individual Contributor, Manager, Director, VP, C-Suite

### AI Configuration
- **system_prompt**: Full system prompt (500-2000 words)
- **persona_id**: Linked to specific persona archetype
- **jtbd_ids**: Array of Jobs-to-Be-Done this agent supports
- **temperature**: LLM temperature (0.0-1.0, typically 0.3-0.7)
- **max_tokens**: Response length limit

### Status
- **status**: 'active', 'draft', 'archived'
- **visibility**: 'public', 'internal', 'private'

## Agent Database Schema

```sql
CREATE TABLE agents (
    id UUID PRIMARY KEY,
    tenant_id UUID REFERENCES tenants(id),

    -- Identifiers
    unique_id VARCHAR(255) UNIQUE,
    name VARCHAR(255) NOT NULL,
    slug VARCHAR(255) UNIQUE,
    domain VARCHAR(255),      -- 'Medical Affairs', 'Regulatory Affairs', etc.
    subdomain VARCHAR(255),   -- 'Field Medical', 'Regulatory Strategy', etc.

    -- Profile
    description TEXT,
    capabilities TEXT[],      -- ['Medical literature synthesis', 'Clinical trial data interpretation', ...]
    expertise_areas TEXT[],   -- ['Oncology', 'Real-world evidence', ...]
    years_of_experience INTEGER,
    seniority_level VARCHAR(50),

    -- AI Configuration
    system_prompt TEXT,       -- Full system prompt
    persona_id UUID REFERENCES personas(id),
    jtbd_ids UUID[],          -- Array of JTBD IDs
    temperature DECIMAL(3,2) DEFAULT 0.5,
    max_tokens INTEGER DEFAULT 2048,

    -- Behavior
    communication_style VARCHAR(255),  -- 'Evidence-based, clear, accessible'
    personality_traits TEXT[],         -- ['Collaborative', 'Scientifically rigorous', ...]

    -- Status
    status VARCHAR(50) DEFAULT 'active',
    visibility VARCHAR(50) DEFAULT 'public',

    -- Metadata
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);

-- Agent capabilities (many-to-many with capability registry)
CREATE TABLE agent_capabilities (
    id UUID PRIMARY KEY,
    agent_id UUID REFERENCES agents(id),
    capability_id UUID REFERENCES capabilities(id),
    proficiency_level VARCHAR(50),  -- 'basic', 'intermediate', 'advanced', 'expert'
    created_at TIMESTAMP DEFAULT NOW()
);

-- Agent knowledge domains
CREATE TABLE agent_knowledge_domains (
    id UUID PRIMARY KEY,
    agent_id UUID REFERENCES agents(id),
    domain_id UUID REFERENCES knowledge_domains(id),
    depth VARCHAR(50),  -- 'surface', 'working', 'deep', 'expert'
    created_at TIMESTAMP DEFAULT NOW()
);

-- Agent prompts (versioned)
CREATE TABLE agent_prompt_versions (
    id UUID PRIMARY KEY,
    agent_id UUID REFERENCES agents(id),
    version INTEGER,
    system_prompt TEXT,
    prompt_starters TEXT[],  -- Example queries to get users started
    active BOOLEAN DEFAULT FALSE,
    created_at TIMESTAMP DEFAULT NOW()
);
```

## Agent Orchestration Patterns

### Pattern 1: Single Agent (Ask Expert Mode 1)
**Use Case**: Quick answer from one expert

```
User Question ‚Üí Agent Selection ‚Üí Single Agent Response
```

**Example**:
- User: "What's the latest on CAR-T therapies for lymphoma?"
- Agent: Oncology Clinical Development Agent
- Response: Evidence-based summary with citations

---

### Pattern 2: Sequential Panel (Ask Panel Type 1)
**Use Case**: Structured expert discussion

```
User Question ‚Üí Moderator Agent ‚Üí Expert 1 ‚Üí Expert 2 ‚Üí ... ‚Üí Moderator Synthesis
```

**Example**:
- User: "Should we proceed to Phase 3 for Drug X?"
- Panel: Moderator + Clinical Dev + Regulatory + Medical + Safety + Finance
- Output: Sequential analysis with final recommendation

---

### Pattern 3: Parallel Exploration (Ask Panel Type 2)
**Use Case**: Brainstorming and innovation

```
User Question ‚Üí Facilitator ‚Üí [Expert 1, Expert 2, ..., Expert N] (parallel)
                           ‚Üí Facilitator Synthesis
```

**Example**:
- User: "What are innovative digital health solutions for diabetes?"
- Panel: Digital Health + Endocrinology + UX + Health Econ + Behavioral Science
- Output: 15-20 diverse ideas clustered into themes

---

### Pattern 4: Adversarial Debate (Ask Panel Type 4)
**Use Case**: Go/no-go decisions

```
User Question ‚Üí Moderator ‚Üí [Advocates] vs [Opponents] ‚Üí Rebuttals ‚Üí Recommendation
```

**Example**:
- User: "Should we acquire Company X for $500M?"
- Advocates: 3 agents arguing FOR
- Opponents: 3 agents arguing AGAINST
- Output: Balanced pro/con analysis with recommendation

## Agent Selection Algorithm

### Mode 1: Manual Selection
**User explicitly chooses agent**

```python
# User selects from agent catalog
selected_agent = user.select_agent(domain="Medical Affairs", role="MSL Expert")
response = agent.query(question=user_question)
```

### Mode 2: Automatic Selection
**AI recommends best 1-2 agents**

```python
# Analyze user question
question_analysis = analyze_question(user_question)
# domain, complexity, urgency, required_expertise

# Score all agents
agent_scores = []
for agent in all_agents:
    score = calculate_fit_score(
        question_analysis,
        agent.expertise_areas,
        agent.capabilities,
        agent.domain
    )
    agent_scores.append((agent, score))

# Select top 1-2 agents
best_agents = sorted(agent_scores, reverse=True)[:2]
responses = [agent.query(user_question) for agent, score in best_agents]
```

### Mode 3: Panel Auto-Assembly
**AI assembles optimal panel of 3-7 agents**

```python
# Analyze question to identify needed expertise
required_expertise = extract_expertise_needs(user_question)
# e.g., ['clinical development', 'regulatory strategy', 'medical affairs']

# Select diverse panel
panel_agents = []
for expertise in required_expertise:
    best_agent = find_best_agent_for_expertise(expertise, exclude=panel_agents)
    panel_agents.append(best_agent)

# Execute panel workflow
panel_results = execute_panel_workflow(
    panel_agents,
    user_question,
    panel_type="structured"  # or 'open', 'socratic', etc.
)
```
